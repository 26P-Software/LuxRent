Module 3: @keyframes Structural Mechanics

Core Use Case: Complex, Multi-Stage Component Keyframing Without JavaScript Controllers

1. Anatomy of an Animation

To implement an animation in CSS, you separate the configuration properties (on the selector) from the actual state checkpoints (inside the timeline definition).

The Shorthand Sub-Properties

  • animation-name: Maps the element to a specific @keyframes timeline block.
  • animation-duration: The total execution runtime clock length for a single complete loop (e.g., 16s).
  • animation-iteration-count: Determines how many cycles run before halting. infinite loops permanently.
  • animation-timing-function: Manages rate curves (e.g., ease-in-out, linear, or customized cubic-bezier arcs).

2. The @keyframes Timeline Map

Think of @keyframes as a timeline calculated from 0% (start) to 100% (end). You declare properties at various percentages, and the browser handles the transitions between them automatically.

The Image Slider Math Formula:

To hold a slide perfectly static and then slide smoothly to the next one, your timeline needs matching percentage steps:

  • 0% - 20%: Frame remains static at translateX(0%).
  • 20% - 25%: Smoothly shifts -25% to slide the next image into view.
  • 25% - 45%: Frame remains static at translateX(-25%).
@keyframes slideSequence {
    0%   { transform: translateX(0%); }
    20%  { transform: translateX(0%); }
    25%  { transform: translateX(-25%); }
    45%  { transform: translateX(-25%); }
    /* Sequence expands continuously per frame */
}
