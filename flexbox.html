Module 1: CSS Flexbox Architecture

Core Use Case: One-Dimensional Component Layouts (Rows OR Columns)

1. The Core Philosophy

Flexbox layout control relies entirely on a parent-child hierarchy. Turning on flexbox changes the rendering behavior of immediate children instantly.

  • Flex Container: The parent element initialized with display: flex; or display: inline-flex;.
  • Flex Items: Universal immediate child elements inside that parent container.
Crucial Axis Rule: Flexbox calculates alignments along two invisible axes:
  • Main Axis: Determined by flex-direction (Defaults horizontally left-to-right).
  • Cross Axis: Runs perfectly perpendicular to your chosen Main Axis.

2. Parent Container Properties

  • flex-direction: row | row-reverse | column | column-reverse;
    Establishes the structural orientation of the main axis.
  • justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;
    Controls asset distribution right along the Main Axis.
  • align-items: flex-start | flex-end | center | baseline | stretch;
    Controls object alignment along the Cross Axis.
  • flex-wrap: nowrap | wrap | wrap-reverse;
    Forces items into single lines or breaks them across multi-tier vectors safely.

3. Child Item Properties

  • flex-grow: [number];: Dictates if an item should expands proportionally to absorb leftover container voids. (Item 2 in demo uses flex-grow: 1).
  • flex-shrink: [number];: Sets structural compression rights if space gets tight.
  • flex-basis: [auto | width];: Configures raw base footprint prior to fluid rendering overrides.
  • align-self: auto | flex-start | flex-end | center | stretch;: Empowers singular targets to cleanly break away from standard structural global parameters. (Item 3 in demo uses align-self: flex-end).
Teaching Tip: Remind students that changing flex-direction to column means justify-content now works vertically, and align-items works horizontally!
